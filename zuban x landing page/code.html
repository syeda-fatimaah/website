Under the Hood

Built With Precision Engineering

ZubanX isn't just polished on the surface — its architecture, security, and tooling are production-grade from day one.

Kotlin (App Language)

The entire Android app is written in Kotlin — Google's official modern language for Android. Think of it as a cleaner, safer Java. It handles everything you see on screen: buttons, lists, animations, navigation.

Primary Language

C++ (Security Engine)

The security layer is written in C++ — a low-level language that runs extremely close to the hardware. It's used here to hide and protect sensitive data like API keys and encryption logic where Java/Kotlin can't reach.

Security Layer

Clean Architecture + MVVM

The app is organized into clear layers: UI, business logic, and data — each separated so changes in one don't break the others. MVVM (Model-View-ViewModel) is the pattern that wires the UI to the data cleanly and safely.

Architecture Pattern

NDK (Native Development Kit)

NDK lets Android apps include C/C++ code alongside Kotlin. It's like giving your app a secret basement that Java code can't access — ZubanX uses it to run the security engine at the lowest possible level.

Native Bridge

JNI (Java Native Interface)

JNI is the bridge between Kotlin and C++. It lets the Kotlin app call functions written in C++ and get results back. Think of it as a translator between two different programming worlds.

Kotlin ↔ C++ Bridge

String Obfuscation & Caesar Cipher

Sensitive strings like API keys are scrambled inside the C++ layer using a cipher — even if someone reverse-engineers the APK, they'll only find encrypted gibberish instead of real values. It's like a code within a code.

Data Protection

Firebase Remote Config

Firebase Remote Config lets us change app settings — like which translation engine to use or what features to show — without publishing a new app update. The app fetches these values at launch from Google's servers.

Remote Configuration

Navigation Component

Android's official Navigation Component manages all screen transitions in one central place, like a GPS for the app. It prevents crashes from double-tapping, handles back stack correctly, and keeps navigation predictable.

Screen Routing

Timber (Logging) & Material Design 3

Timber is a smarter logging library — it only shows debug messages during development, keeping the production app clean. Material Design 3 is Google's latest design system providing the components, colors, and spacing used across the UI.
